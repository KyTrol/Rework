uDj

RevolutionUC Hackathon

RevolutionUC is a student organized hackathon held at the University of Cincinnati. Personally, this was the second hackathon that I participated in. My group went in with an idea in mind, and we were able to execute on our plan. The idea was to create an Internet of things jukebox, dubbed uDj, that could be accessed by multiple clients simultaneously through a web application. This moves the control of the song queue from an individual to the group. Multiple users can add songs to a single queue which would then combine these song requests to play back.

This web application uses HTML, CSS, and JavaScript for the interface and jQuery to communicate to the back-end. The back-end server is coded in Node.js and uses Python along with RabbitMQ to handle the song queue. I created the interface for this application and contributed to handling the requests from the server. I had a lot of fun over the course of this hackathon and learned a lot about implementing front-end logic to interact with the back-end.

Universify

DerbyHacks Hackathon

Held at the University of Louisville, DerbyHacks was the first hackathon that I experienced. I traveled to Louisville with a friend, Dom, and we hoped to find other students to join up with. Miraculously, we found two other students looking to do just that. The idea we brainstormed was to create Universify. A web application where students could log in and ask questions directed at specific classes they were enrolled in. They could also log in and answer questions other students at their university had asked. I collaborated with Dom to create the interface and communication on the front-end, facilitated by AngularJS. The other two members coded the back-end using Python Flask. This was my first encounter with a front-end framework and how data is transmitted from the server to the application.
